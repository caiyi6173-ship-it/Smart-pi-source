# SmartTCM

SmartTCM 是一个以树莓派为核心运行环境的中医智能助手系统，组合了边缘视觉推理、本地语音交互、OpenClaw 智能体协同，以及中医知识库 RAG 服务。

这个目录是从树莓派实际运行环境中回收并整理出来的源码快照，目标是把它变成适合 GitHub 管理的工程仓库。当前重点保留：

- 核心业务代码
- 部署脚本
- 配置模板
- OpenClaw 集成资产
- RAG 应用逻辑

不打算直接纳入仓库的内容包括模型权重、运行缓存、日志、采集图片和视频、数据库状态等运行时产物。

## 模块概览

- `edge/`
  树莓派边缘控制层，包含语音代理、边缘桥、传感器中枢、MJPEG 流服务、启动脚本和 systemd 服务文件。

- `rag-chroma/`
  当前主用的 RAG 服务，使用 Chroma 作为本地持久化向量后端，向 SmartTCM 其他组件提供知识检索和问答 API。

- `config/`
  边缘桥、语音代理、流服务、传感器服务等运行配置。上传 GitHub 时应以模板文件为主，避免提交真实配置。

- `openclaw/`
  SmartTCM 侧的 OpenClaw 集成脚本、技能说明和相关项目资产。

- `openclaw-runtime/`
  从树莓派 OpenClaw 运行环境中回收的已安装技能内容，可用于对照项目技能和运行态技能是否一致。

- `openclaw-workspace/`
  从 OpenClaw 工作区提取的公开文档，用于理解智能体工作区约定和行为上下文。

- `yolo_clean/`
  从树莓派回收并过滤后的 YOLO 相关源码目录，已经去除了虚拟环境、模型权重和明显的运行时大文件。

- `yolo-prj/`
  树莓派桌面上的一个小型 YOLO 试验目录，可选保留。

## 当前未纳入仓库的内容

树莓派原始环境中还有很多运行态文件，它们不适合直接公开到 GitHub：

- `SmartTCM/models/` 下的模型文件
- HuggingFace 与 TTS 缓存
- 相机采集图片和视频
- 各种运行日志与推理结果 JSON
- 本地虚拟环境
- `.env` 和真实密钥
- OpenClaw 的记忆库、会话库、设备身份状态

## 推荐的仓库方向

如果这里要作为最终的 GitHub 主仓库，建议按下面方向继续整理：

1. 把 `config/` 下的真实配置文件替换成脱敏后的 `*.example` 模板。
2. 明确是否只保留 `rag-chroma/` 作为主 RAG 服务，还是同时保留旧版 `rag/`。
3. 再清理一轮 `yolo_clean/`，只保留仍然和 SmartTCM 主线相关的 YOLO 代码。
4. 增加一份顶层部署文档，说明树莓派部署步骤、模型准备和服务启动顺序。
5. 维持顶层 `.gitignore`，避免误提交运行数据和敏感文件。

## 系统高层流程

1. 边缘视觉模块采集图像或视频帧，并执行舌象相关推理。
2. 边缘桥向本地组件暴露硬件控制、状态读取和动作执行能力。
3. 语音代理处理唤醒、录音、语音识别、播报和设备命令调度。
4. OpenClaw 可作为更高层的智能体协调层，负责决策和动作标记。
5. RAG 服务负责中医知识检索、重排序、引用组织和答案生成。

## 说明

- 当前快照来自真实运行中的树莓派环境，因此不少脚本仍然带有 `/home/pi/SmartTCM/...` 这类部署路径。
- 公开发布前应优先使用脱敏模板，而不是直接提交真实配置。
- 目前建议把 `yolo_clean/` 视为可整理的源码候选目录，而不要使用旧的脏恢复目录。
